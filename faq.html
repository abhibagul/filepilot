Frequently Asked Questions

Everything you need to know about FilePilot's encryption architecture, deployment models, and features.

1. What is FilePilot and how are the two components related?

FilePilot is an open-source project consisting of a high-speed desktop client (Tauri/Rust) and an optional centralized provisioning server (Node.js/Express) called the Enterprise Vault.

You can run the Desktop Client standalone to manage local and remote connections. If you work in a team, you can link the Desktop Client to an Enterprise Vault server to sync connection profiles, automate backups, log security audit events, and enforce security policies.

2. How does Zero-Knowledge local encryption protect my passwords?

When you launch the client, you define a local Master Password. This password is run through 600,000 iterations of PBKDF2 to derive a strong encryption key. This key is used to seal your connection credentials (passwords, private SSH keys, cloud keys) at rest using AES-256-GCM envelope encryption.

The master password is never stored on disk and never sent to any external server. Without it, your connection configuration remains completely encrypted, protecting you even if the local database file is extracted from your machine.

3. How does the Enterprise Vault distribute credentials safely?

When using the Enterprise Vault, raw passwords are encrypted in the central database using AWS KMS, Azure Key Vault, Google Cloud KMS, or HashiCorp Vault.

When a Desktop Client requests synchronization, the server validates the client's token, audits the access log, and verifies IP access rules. If cleared, the encrypted secrets are transferred over a secure TLS WebSocket connection, decrypted in-memory on the client, and immediately sealed into the client's local database using the client's master key. Raw passwords are never visible to the developer in transit or in the UI.

4. What databases can I run the Enterprise Vault server on?

FilePilot Enterprise Vault is designed with runtime database configuration. You can switch between SQLite, PostgreSQL, and MySQL backends directly from the system admin panel. Database configuration settings are verified using a connection tester wizard before committing.

5. How does KEK Rotation prevent downtime or data corruption?

The Vault Server uses envelope encryption: a Master Key Encryption Key (KEK) wraps individual Data Encryption Keys (DEKs) for each group, which then encrypt individual connection profiles.

When KEK rotation is triggered, the server decrypts the Group DEKs with the old KEK and wraps them under the new KEK. The actual profile credentials are never modified, avoiding any performance degradation or data corruption.

6. What is a blockchain-style audit trail?

Every transaction logged in the Enterprise Vault features a SHA-256 signature chain. Each entry contains a hash of its own metadata combined with the hash of the preceding entry (prev_hash).

The admin panel provides a validation task that scans the log table. If a malicious attacker edits or deletes a row, the cryptographic signature chain is broken, pointing to the exact row that was modified.

7. How do JavaScript macros automate transfer workflows?

The Desktop Client includes a sandboxed JavaScript runtime environment. You can write custom scripts using a built-in API (fp.copy(), fp.list(), fp.rename(), fp.delete(), fp.notify()) and trigger them on app launch, when connecting to a profile, or when disconnecting.

8. Can I run FilePilot Client without the Enterprise Vault?

Yes, stand-alone mode is fully supported. By default, the Desktop Client acts as a self-contained manager storing profiles inside a local sealed SQLite file.

9. Does FilePilot collect telemetry or transfer usage logs?

Never. We have zero telemetry, zero analytics tracking, and no external calls. All configurations and operations are processed on your workstation or inside your private Enterprise Vault server.

10. How does the folder synchronization resolve file differences?

The folder sync module compares directory structures on both sides. If the same file was modified on both local and remote nodes, a conflict resolution modal launches, showing side-by-side snippet previews so you can decide which version to retain.

11. How are SSH private keys and passphrases secured by FilePilot?

SSH keys are loaded, encrypted via AES-256-GCM using your PBKDF2 derived master key, and written to the sealed local database. During transfer loops, keys are decrypted directly in-memory and passed to the SSH authentication channel. Raw key files are never written to temporary folders.

12. How does the Plugin SDK guarantee system security?

Plugins must declare all network and filesystem permissions within a static plugin.json manifest. The Tauri core enforces sandbox isolation based on these declarations, preventing unauthorized plugins from accessing local credentials or making external network requests.
